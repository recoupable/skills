# FaceTime-Call Ads with Grok Imagine 1.5

Two people on what looks like a screen-recorded FaceTime call, talking about the product.
It works because it breaks the feed pattern — not a talking head, not a podcast clip — and
because a recorded call between friends still reads as native. Every clip is one
`xai/grok-imagine-video/v1.5/image-to-video` generation: **dialogue written in the prompt is
spoken verbatim, with the speaker's own voice and correct lipsync**. No TTS, no lipsync pass —
both were bake-off losers (a lipsync pass deadens the face; HeyGen avatars read as studio ads).

Reference build: `strategy/recoup-rostrum-baseline/sweetman/content/facetime-call/`
(NOTES.md = full decision log; `index.html` = the hyperframes FaceTime chrome; final:
`renders/facetime-call_*.mp4`). `FAL_KEY` lives in `mono/api/.env.local`.

## ⚠️ Consent gate (before anything else)

Real artists' AI likenesses speaking words they never said **never ship without the artist's
explicit sign-off**, and posts carry an AI + permission disclosure. Build the video, send it
to each artist privately for approval (it doubles as the best warm-lead touch they'll get),
and only then post. Fictional characters need no gate.

## The pipeline

1. **Script** the call (~45-60s) as alternating turns: A, B, A, B. Topic-first, one concrete
   proof point, an organic close ("okay, send me the link") — see ship-posts' principles.
2. **Faces**: one 9:16 "FaceTime frame" image per character (below).
3. **Takes**: one Grok generation per turn. QC each with Whisper + frame pulls.
4. **Cut**: concat the takes — every cut is a character switch.
5. **Chrome**: hyperframes FaceTime UI on top. 6. **Ship**: ship-posts skill.

## Take structure (works WITH the model's limits)

- **15s is a hard cap per generation** (enqueue accepts more; generation rejects it). The model
  works in ~5s internal blocks — prompt in timestamped beats: `(0-5s) … (5-10s) … (10-15s) …`.
- **One generation = one continuous voice.** Voice is NOT stable across generations, so never
  split one character's consecutive lines across takes. Hide every take joint behind a cut to
  the other character. Alternating 15s turns does this for free.
- Need one unbroken shot >15s? `xai/grok-imagine-video/extend-video` (+2-10s per call,
  "what happens next" prompt) continues voice and scene — but input must be **≤15.0s**
  (Grok outputs run ~15.04s; trim to 14.9s first). Verified to 25s word-perfect.
- Size each turn's duration to its lines (~2.5-3 words/sec). Dead air at the end of a take is
  normal — find the speech end from the Whisper word timestamps and trim in assembly.

## Prompt grammar (each item is a shipped-and-burned lesson)

Template that produced word-perfect takes:

```
FaceTime video call, subtle camera micro-shake, <room/lighting>, natural casual delivery
with clear enunciation. Exactly one person in the room, only <his/her> voice, <his/her> lips
move with every word <he/she> speaks starting immediately from the very first word.
(0-5s) <The character> says, speaking from the first moment: "<line>"
(5-10s) <small physical direction> and says: "<line>"
(10-15s) Looking straight into the camera <he/she> says: "<line>"
```

- **Never put an action verb in the dialogue beat** — "laughs and says" gets you 3 seconds of
  laughing INSTEAD of the line. Emotion goes in the tone direction ("warm amused tone");
  the beat verb is always plain `says`.
- "lips move with every word … from the very first word" — else takes open with miming/smiling.
- "Exactly one person in the room, only her voice" — kills phantom off-screen voices.
- "She holds the phone with one hand the entire time, her other hand stays out of frame" —
  kills third-hand artifacts.
- **QC every take**: extract audio (`ffmpeg -vn`), transcribe (`fal-ai/whisper`,
  `chunk_level: "word"` for trim points), diff against the script; pull 2-3 frames at
  suspect timestamps. Slurred words ("city by cit") and mime-gaps mean re-roll (~$2/take).

## Faces — the identity pipeline

Fictional character: one `xai/grok-imagine-image` text-to-image ("vertical smartphone selfie
video call still frame … slightly grainy front-camera quality … authentic FaceTime screenshot
aesthetic, not professional photography").

Real person (after the consent plan): source photo → `xai/grok-imagine-image/edit`:
1. **Letterbox square sources to 9:16 BEFORE the edit** (ffmpeg pad) — passing a square image
   with `aspect_ratio: 9:16` stretches the face (x-axis squish).
2. Edit prompt: "The same person from the reference photo (<identity anchors: hair, gauges,
   tattoo, clothing>) … seen through a smartphone front camera … **Plain raw camera image
   only: no user interface, no status bar, no text, no icons**" — without that clause Grok
   bakes a fake (misspelled) FaceTime UI into the image.
3. Bars usually survive → one more edit: "Extend this exact scene to fill the entire vertical
   frame … keep the person exactly as is. No black bars."
4. Strip AI-gibberish clothing text with a targeted edit if it appears.
5. Reuse the SAME final frame for all of that character's takes — that is what locks face and
   room across generations.

Artist photos: check the workspace first (`sweetman/content/*/assets/art/`), then Recoup
socials (`avatar` is only 150px; full-size comes from the og:image of their IG **post** pages),
then Spotify oEmbed (`open.spotify.com/oembed?url=<artist url>` → `thumbnail_url`).

## fal mechanics

- Queue: `POST https://queue.fal.run/<model>` → poll `…/requests/{id}/status` → GET
  `…/requests/{id}`. Auth `Authorization: Key $FAL_KEY`.
- Media inputs go through fal storage (`POST https://rest.fal.ai/storage/upload/initiate`
  → PUT bytes → use `file_url`); data-URIs over ~700KB are rejected by some models.
- Validation failures (>15s duration, oversized payloads) cost nothing.

## Assembly + chrome + ship

- Normalize each take (`720:1280`, 30fps, h264 crf20, aac 48k) → concat demuxer. Trim each
  take at its Whisper speech-end (+0.3s).
- Chrome is a hyperframes composition (copy the reference project): full-bleed muted `<video>`
  + same-file `<audio>`, caller-name bar + ticking duration (deterministic per-second
  `tl.call`s), red screen-record pill, PiP showing the OTHER character (their frame image,
  slow 1.0→1.05 drift, crossfade swap at every cut), control pill, SVG-turbulence grain,
  end card. Geist fonts — the renderer cannot supply Apple system fonts.
- Then `npm run check` (zero errors) and `npm run render`. Publish + measure via
  `recoup-internal-social-ship-posts`.

## Cost (measured 2026-07-02)

Grok 1.5 video bills **$0.14/s at 720p** ($0.08/s at 480p — untested quality lever); identity
edits $0.022/image; Whisper pennies. A debugged 50s two-character video ≈ **$10**; first-time
exploration incl. bake-offs and QC re-rolls ran ~$32. Budget ~2 re-rolls per 4 takes.
